## Why

The org diagram's cross-repo navigation links are broken on GitHub. `docs/architecture.md` in an
instance repo links to each repo's own diagram via `docs/{repo}/architecture.md`, but that file
already lives inside `docs/`, so GitHub resolves the relative link against its own directory and
produces `docs/docs/{repo}/architecture.md` — a 404. Confirmed live on
`industrial-curiosity/panopticon-test`: every link in `docs/architecture.md` to a child repo's
diagram 404s. The existing test only asserts the link text contains the right substring, never
that it resolves correctly relative to where the file lives, so this has been broken since the
diagram feature's original implementation and gone undetected across every later "arch linking"
pass.

## What Changes

- Fix `panopticon/diagrams.py` to emit `{repo}/architecture.md` (no `docs/` prefix) for both the
  per-repo section's own-diagram link and each table row's other-repo link, since
  `docs/architecture.md` already lives inside `docs/` and the href must be relative to that.
- Strengthen `tests/test_diagrams.py`'s navigation-link test to assert the literal correct href
  (`svc-a/architecture.md`, not `docs/svc-a/architecture.md`) so a regression back to the buggy
  form fails the test instead of passing it.
- Tighten `openspec/specs/architecture-diagrams/spec.md`'s scenario wording so it states the
  literal relative href text the link must use, not just the absolute path it resolves to — the
  current wording ("leads to `docs/{repo}/architecture.md` in the instance repo") is true as a
  description of the resolved target but reads as an instruction for the href itself, which is
  what the implementation did literally.

## Capabilities

### New Capabilities

(none)

### Modified Capabilities

- `architecture-diagrams`: the org diagram's per-repo section back-link and per-row table link
  must use `{repo}/architecture.md` as the literal relative href (not `docs/{repo}/architecture.md`),
  since the org diagram file itself already resides at `docs/architecture.md`.

## Impact

- `panopticon/diagrams.py` — `render_org_diagram()`'s per-repo section link (currently line 139)
  and `_table()`'s per-row link (currently line 115).
- `tests/test_diagrams.py` — `test_navigation_links_to_child_repo_docs`.
- `openspec/specs/architecture-diagrams/spec.md` — the two scenarios describing link hrefs.
- Every existing instance repo's `docs/architecture.md` is stale until regenerated by the next
  merge to a child repo's default branch (master-sync capability); no migration script needed
  since the file is fully regenerated, not incrementally patched.
